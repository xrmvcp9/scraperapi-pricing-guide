## 先说说 ScraperAPI 是干嘛的

简单讲，ScraperAPI 是一个把"爬虫这件麻烦事"打包成一个 API 调用的服务。你发一个请求过去，它在背后帮你处理代理 IP 轮换、JavaScript 渲染、验证码绕过、请求重试这些脏活累活，返回给你的就是一段干净的 HTML（或者结构化的 JSON，如果你抓的是亚马逊、谷歌这类常见目标）。

它家所有套餐都标配了这几样东西，不分高低档：

- JS 渲染（处理动态加载的网页）
- 高级代理池（轮换 IP，避免被封）
- 自动重试机制
- 不限带宽
- 99.9% 的正常运行时间保证

差别主要在「额度多少」「并发线程数」和「地理定位精度」这几个维度上。下面具体拆开讲。

## ScraperAPI 的计费逻辑：先搞懂"信用点"是什么

在看价格表之前，有一件事必须先弄明白：ScraperAPI 不是按"请求次数"收费的，而是按 **API Credits（信用点）** 收费。

每次请求消耗多少信用点，取决于你抓取的目标和用到的功能：

- 普通静态页面、不需要 JS 渲染：消耗较少，通常 1 个信用点起
- 需要 JS 渲染的动态页面：额外加消耗
- 亚马逊、谷歌、Walmart 这类有结构化解析的目标：消耗会更高，单次请求可能到 5～75 个信用点不等，具体取决于目标网站的反爬强度

这意味着同样是「100,000 信用点」，如果你抓的是普通博客页面，可能能跑十万次请求；但如果你抓的是带 JS 渲染 + 高强度反爬的电商页面，实际可用次数会大打折扣。这是很多新手在选套餐时容易忽略的一点——**别只看信用点数字，先估算你的目标网站大概会消耗多少信用点一次**，再倒推该选哪个套餐。

## 各档套餐到底该怎么选？

价格表摆出来了，但数字本身不会替你做决定，下面按使用场景拆一下。

### 刚接触爬虫 / 只是想测试一下：Free 版

如果你还没决定要不要长期用 ScraperAPI，先别急着掏钱。免费版每月有 1,000 个信用点，最多 5 个并发连接，新注册的账号头 7 天还能额外拿到 5,000 个测试用的信用点。这个量级足够你把官方文档里的几种集成方式（Python、Node.js、PHP、Ruby、Java 都有现成的库）都跑一遍，看看返回结果是不是你想要的。

### 个人项目或者副业：Hobby（$49/月）

Hobby 是付费套餐里最便宜的一档，10 万信用点、20 个并发，地理定位限定在美国和欧盟。如果你是个人开发者，抓的是博客、论坛、中小型电商站这类反爬强度一般的目标，这一档基本够用。

需要提醒的是，多个第三方评测都提到一个共性问题：Hobby 这一档的「起步门槛」相对其他同类竞品偏高，而且如果请求被目标网站拦截，有些情况下被拦截的请求仍然会计费，所以实际可用次数往往比信用点面值看起来要少。建议先用免费额度估算一下你的目标网站平均每次请求要花多少信用点，再决定要不要升级。

### 团队协作、稳定业务流：Startup 到 Business

从 Startup（$149/月，100 万信用点）往上，地理定位还是停留在美国和欧盟，但到了 Business（$299/月，300 万信用点）开始，地理定位升级为全球国家级定位，并且解锁了「无限分析历史」——也就是说你能查看的历史抓取数据不再有时间限制。如果你的业务需要长期监控某些站点的数据变化（比如电商比价、市场研究），这个功能挺实用的。
